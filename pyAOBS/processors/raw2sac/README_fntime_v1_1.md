# fntime_v1_1.py

Python实现版本的 `fntime_v1.1.c`，用于解析OBS文件名中的时间戳信息。

## 功能

从OBS设备生成的十六进制文件名中提取日期时间信息，计算精确的时间戳，并分析文件大小和数据点数。

## 使用方法

```bash
python fntime_v1_1.py fileNameList TC sps_pre
```

### 参数说明

- **fileNameList**: 输入文件名列表文件（每行一个文件名，格式：12字符十六进制 + 换行）
- **TC**: 时钟参数（从A*.LOG文件读取）
- **sps_pre**: 预设采样率（Hz）

### 示例

```bash
python fntime_v1_1.py file_list.txt 524288.0 125.0
```

## 输入文件格式

文件名列表文件格式：
```
51A6D1595DD
<换行或空行>
51A6D15A0E1
<换行或空行>
...
```

每行最多12个十六进制字符，下一行会被跳过（对应C代码的`fgets(linefd,5,fpin)`）。

## 输出

生成 `fntime.txt` 文件，包含每个文件的时间戳信息和统计数据：

```
51A6D1595DD  2020-06-19 13:05:25.123 UTC time_length=10.500000 second, delta=8.333333 msec, Gap=0.000000 msec
51A6D15A0E1  2020-06-19 13:05:35.623 UTC time_length=10.500000 second, delta=8.333333 msec, Gap=0.000000 msec
...
```

## 核心算法

### 1. 十六进制转换
- `hexto_dec()`: 使用加权方式将十六进制字符串转换为十进制整数
- 对应C代码: `HextoDec(unsigned int a0, unsigned char *p)`

### 2. 日期时间解析
- 从前8个十六进制字符中提取年、月、日、时、分、秒
- 使用位运算提取各部分（与C代码完全对应）

### 3. 毫秒计算
- 从最后3个十六进制字符解析毫秒
- 使用公式: `nzmsec * 4096 / PCIk`（其中 `PCIk = TC / 256.0`）

### 4. 文件类型判断
- 根据第一个十六进制字符的高3位判断：
  - `date[0] >> 3 == 0`: 4通道数据（12字节/点）
  - `date[0] >> 3 == 1`: 3通道短周期数据（9字节/点）

## 与C版本的区别

1. **错误处理**: Python版本使用异常处理，而不是直接`exit(0)`
2. **可读性**: 函数化设计，代码更易读和维护
3. **扩展性**: 易于添加新功能和单元测试
4. **兼容性**: 输出格式与C版本完全一致
